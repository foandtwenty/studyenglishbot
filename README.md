# Study English Bot

Телеграм-бот для тренировки английского через флеш-карточки с интервальным
повторением. Ошибочные карточки возвращаются через 2–3 хода и ещё раз в конце
сессии — так слова запоминаются лучше.

## Типы тренировок

| Тип | Что тренирует | Карточек |
|-----|---------------|----------|
| 🔤 Неправильные глаголы | вспомнить V2 и V3 (кнопки или ввод текста) | 97 |
| 📍 Предлоги | in / on / at в контексте | 43 |
| ➕ Глаголы + to / -ing | какой шаблон у глагола | 51 |
| 🔗 Прилагательные + предлог | afraid **of**, nervous **about**… | 58 |

На каждом ответе показывается пример употребления и грамматическое правило
(📖), чтобы было понятно, *почему* именно такая форма.

## Возможности

- Прогресс-бар и интервальное повторение внутри сессии
- 🟢🟡🔴 **Уровни сложности** — Базовый / Средний / Продвинутый (глаголы
  тиражированы по частотности, остальное — по употребимости правила;
  разметка в `levels.py`)
- 🔔 **Тренировка дня** (Leitner: 1→2→4→7→15→30 дней) — до 30 карточек на
  повтор (сложные первыми) + до 7 новых в день; кнопка «Тренировка дня (N)»
  на главном экране
- 🎲 **Всё вперемешку** — интерливинг карточек всех типов (эффективнее
  блочной практики)
- 🔔 **Ежедневные напоминания** о due-карточках (время и часовой пояс
  настраиваются; опт-аут в «Помощи»)
- 🎯 Режим «только ошибки» — тренировка персонально сложных карточек
- ✏️ Режим ввода для глаголов (печатать V2/V3 вместо кнопок); на экране
  результата любой отправленный текст работает как «Дальше»
- 🔄 **Обратный режим RU→EN** — показывается перевод, вспоминаешь глагол
  и его формы (более глубокое запоминание)
- Прощение опечаток: одна опечатка в слове от 5 букв — «верно, но сверь
  написание» (перепутанные формы, например came↔come, не прощаются)
- ↩️ Отмена последнего ответа на экране результата
- 📅 На верном ответе показывается интервал следующего повторения
- 💡 Подсказка: первая буква и длина V2
- Статистика: серия дней, освоено/изучается, история сессий
- Весь UI живёт в одном редактируемом сообщении — чат не засоряется

## Управление

Полностью на кнопках — отдельных команд в интерфейсе нет. `/start` запускает
бота и возвращает в главное меню (на случай, если карточка потерялась в чате).
`/admin` — скрытая сводка по боту для `ADMIN_ID`. Прочие команды (`/stats`,
`/help` и т. д.) работают как тихий фолбэк, но в меню не показываются.

## Запуск

```bash
pip install -r requirements.txt
export BOT_TOKEN="<токен от @BotFather>"
python bot.py
```

## Переменные окружения

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `BOT_TOKEN` | токен бота (обязателен) | — |
| `DB_PATH` | путь к SQLite-базе | `study_english.db` |
| `STATE_PATH` | файл состояния сессий (pickle) | рядом с `DB_PATH` |
| `ADMIN_ID` | Telegram-id для `/admin` | — |
| `REMINDER_HOUR` | час ежедневных напоминаний (время сервера, обычно UTC) | `18` |

## Деплой (Railway)

Бот запускается как `worker` (`python bot.py`, long-polling).

> ⚠️ **Подключите Volume.** Файловая система Railway эфемерна: без тома при
> каждом редеплое теряются и база (`DB_PATH`), и состояние сессий
> (`STATE_PATH`). Смонтируйте том (например, на `/data`) и задайте
> `DB_PATH=/data/study_english.db` — файл состояния по умолчанию ляжет
> в тот же каталог, так что активные сессии и кнопки переживут перезапуск.

## Тесты

```bash
pip install -r requirements-dev.txt
pytest
```

119 тестов покрывают целостность контента (дубли id, коллизии, варианты
ответов), механику сессий и интервального повторения, слой БД (серии по дням,
статистика) и сборку экранов. Сеть не нужна — Telegram заменён на заглушку.
Гоняются автоматически на каждый push (GitHub Actions).
